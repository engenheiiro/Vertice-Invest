📘 Vértice Invest - Cheat Sheet de Comandos (Oficial)

🚀 1. Setup & Inicialização

# Instala todas as dependências (Raiz, Client e Server) de uma vez
npm run setup

# Inicia o modo de desenvolvimento (Frontend + Backend + Watchers)
# Conecta ao MongoDB definido no .env
npm run dev

# Inicia o servidor em modo produção (sem nodemon, sem HMR)
npm start

⚙️ 2. Operações de Banco de Dados & Manutenção

# [IMPORTANTE] Sanitização e Correção
# Corrige Tickers errados (Typos) e atualiza metadados (Setores) sem apagar histórico.
# Use isso se notar ativos duplicados ou nomes errados.
npm run migrate:fix

# [PERIGO] Reset de Mercado
# ⚠️ APAGA todos os ativos do banco e recria a lista base limpa.
# Use apenas se o banco estiver corrompido ou para iniciar do zero.
npm run seed:market

# Gestão de Acesso
# Promove um usuário existente para ADMIN (Acesso total + Debug).
npm run seed:admin seu@email.com

# Lista todos os usuários cadastrados no banco (nome, email, plano, status, role, validade).
npm run list:users

# Altera o plano de um usuário (GUEST | ESSENTIAL | PRO | BLACK).
# Dias é opcional — padrão: 365. Define status como ACTIVE automaticamente.
npm run set:plan seu@email.com PRO
npm run set:plan seu@email.com PRO 30

# Redefine a senha de um usuário (mínimo 6 caracteres).
npm run set:password seu@email.com NovaSenha123

📡 3. Inteligência & Dados (Produção/Rotinas)

# Protocolo V3 Completo (Heavy Sync)
# 1. Baixa cotações e indicadores macro.
# 2. Executa Scraper do Fundamentus.
# 3. Roda IA para gerar Ranking e Valuation.
# 4. Executa Scanner do Radar Alpha.
npm run sync:prod

# Roda APENAS o Scanner do Radar Alpha + Backtest dos sinais ativos + Auditoria de precisão.
# Use quando quiser forçar uma varredura manual sem rodar o sync completo de mercado.
# Equivalente ao que o cron executa automaticamente a cada 15 min (2,17,32,47 de cada hora).
npm run scan:radar

# Gera relatório completo do Radar Alpha em reports/radar_latest.txt.
# Mostra sinais ativos, histórico recente (48h), win rate, e diagnóstico detalhado de
# por que cada classe (Ações BR, FIIs, STOCK_US) passa ou não passa nos filtros do scanner.
npm run radar:report

# Busca o histórico de preços, calcula SMA200, EMA50, Volatilidade (anualizada) e Beta (usando o IBOV como benchmark), e salva.
# É utilizado pelo comando sync:Prod para montar a análise/ranking.
npm run sync:TimeSeriesWorker

🔁 4. Backtest de Portfólio

# Simula "e se eu tivesse comprado esse portfólio X tempo atrás?"
# Interativo: escolha o usuário, o ranking (Brasil 10, Ações, FIIs, Cripto),
# o perfil de risco (Defensiva / Moderado / Arrojado), a data e o valor a investir.
# Busca os preços históricos de cada ativo na data escolhida, cadastra as transações
# na carteira do usuário e reconstrói os snapshots — exibindo o P&L real desde aquela data.
npm run backtest:portfolio

🧪 5. Testes & Qualidade (QA)

# Testes Unitários do Backend (Serviços, Models, Utils)
npm run test:server

# Roda TODOS os testes do servidor de uma vez (74 testes — QA completo)
npm run test:all

# Testes de Carteira (Snapshot, TWRR, Circuit Breaker, Sharpe, FIXED_INCOME)
# Valida calculateDailyDietz, acumulação de cota, anti-reset e cálculo live de KPI.
npm run test:wallet

# Testes Unitários do Frontend (Componentes, Hooks)
npm run test:client

# Teste de Regressão Financeira (Matemática Crítica)
# Verifica se o cálculo de TWRR, Cotas e Juros Compostos está batendo com o gabarito.
npm run test:regression

# Teste de Robustez do Scraper (Ingestão)
# Verifica se o sistema aguenta dados sujos vindos do Fundamentus.
npm run test:ingestion

# Teste de Pegar 5 ativos da época e comprar o preço
npm run test:backtest

# Testa o scraper do Fundamentus diretamente (verifica se o site mudou estrutura)
npm run test:fundamentus

# Verifica erros de lint no Client e no Server
npm run lint

📦 6. Build (Deploy)

# Gera os arquivos estáticos otimizados do React para produção
npm run build

# Valida o backend (node --check em todos os arquivos-fonte) — gate de sintaxe/import.
# Backend é Node ESM puro (sem bundle); este comando pega erro antes do deploy.
npm run build:server

# Build completo: valida o server + gera o build do client.
npm run build:all

# Typecheck do client (tsc) + validação do server. Roda no CI.
npm run typecheck
