raciocínio financeiro/arquitetural, ou envolver dinheiro/segurança,
promova para Opus. Nunca use Opus para "adicionar 10 testes iguais" ou
"colocar aria-label".


================================================================
COMO VERIFICAR QUE CADA ONDA FUNCIONOU (testes de ponta a ponta)
================================================================
1. Backend: rodar "npm run test:ci" (na raiz e em /server) — testes verdes.
2. Frontend: rodar "npm test" em /client — testes passando.
3. Lint/tipos: "npm run lint" (ESLint) e "npm run lint" do client
   (tsc --noEmit) sem erros.
4. App de verdade: subir com "npm run dev" na raiz e validar:
   login -> carteira -> adicionar ativo -> research -> logout.
5. CI: abrir um PR e confirmar pipeline (lint -> typecheck -> test ->
   build) verde.
6. Acessibilidade/Performance: rodar Lighthouse/axe nas telas principais
   e comparar a nota antes/depois.
